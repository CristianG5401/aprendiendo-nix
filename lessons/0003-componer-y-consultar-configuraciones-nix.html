Lección 0003 · Composición de configuraciones

Componer y consultar configuraciones Nix

Objetivo: combinar defaults y overrides de un home lab, y consultar atributos opcionales sin romper la evaluación.

Tu misión en esta lección

Partirás de una configuración base y aplicarás cambios locales. Al terminar podrás explicar por qué gana cada valor y cómo ofrecer defaults seguros.

Abre la referencia rápida de operadores.

Recuperación: ++

Sin ejecutar Nix, predice el resultado de [ "git" "curl" ] ++ [ "jq" ].

[ "git" "curl" "jq" ]. ++ concatena listas y conserva su orden.

1. //: defaults más overrides

let
  baseConfig = {
    hostname = "homeserver";
    tools = [ "git" "curl" ];
  };

  localOverrides = {
    hostname = "homeserver-prod";
    tools = [ "jq" ];
  };
in
  baseConfig // localOverrides

El resultado contiene los atributos de ambos sets. Si un nombre aparece en los dos, gana el valor del lado derecho.

Analogía JavaScript/TypeScript:

const effectiveConfig = {
  ...baseConfig,
  ...localOverrides,
};

Límite importante: // es superficial. Si ambos lados tienen un atributo monitoring que contiene otro objeto, el valor derecho reemplaza todo monitoring; no mezcla recursivamente sus atributos internos.

2. ?: preguntar si existe

effectiveConfig ? monitoring
effectiveConfig ? backupTarget

? devuelve un boolean: true si el attribute path existe y false si no existe. No devuelve el valor del atributo.

3. or: seleccionar con fallback

effectiveConfig.backupTarget or "/srv/backups"

Esta expresión intenta seleccionar backupTarget. Si no existe, produce "/srv/backups" en lugar de abortar la evaluación.

or no es el operador booleano OR; forma parte de attribute selection. El OR booleano de Nix se escribe ||.

Ejercicio: configura tu home lab

Abre lab/lesson-0003/operators.nix y predice sus seis atributos antes de evaluar:

let
  baseConfig = {
    hostname = "homeserver";
    tools = [ "git" "curl" ];
    monitoring = {
      endpoint = "/metrics";
    };
  };

  localOverrides = {
    hostname = "homeserver-prod";
    tools = [ "jq" ];
  };

  effectiveConfig = baseConfig // localOverrides;
  extraTools = [ "htop" ];
in
{
  hostname = effectiveConfig.hostname;
  tools = effectiveConfig.tools ++ extraTools;
  hasMonitoring = effectiveConfig ? monitoring;
  hasBackupTarget = effectiveConfig ? backupTarget;
  monitoringEndpoint = effectiveConfig.monitoring.endpoint or "/health";
  backupTarget = effectiveConfig.backupTarget or "/srv/backups";
}

Dentro del contenedor del laboratorio, evalúa:

cd /workspace/lab/lesson-0003
nix-instantiate --eval --strict operators.nix

Resultado esperado:

{ backupTarget = "/srv/backups"; hasBackupTarget = false; hasMonitoring = true; hostname = "homeserver-prod"; monitoringEndpoint = "/metrics"; tools = [ "jq" "htop" ]; }

Comprueba tu modelo mental

¿Por qué tools no conserva git y curl?

Porque // reemplaza superficialmente el atributo tools; después ++ añade htop a la lista reemplazante [ "jq" ].
